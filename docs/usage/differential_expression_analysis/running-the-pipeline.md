# Run nf-core/rnaseq for differential expression

RNA-seq measures transcript abundance across samples. The nf-core/rnaseq pipeline handles the computational steps from raw FASTQ files to quantified gene counts: quality control, adapter trimming, alignment, and quantification. The output includes a DESeq2 object that you can load directly into R for differential expression analysis.

## Experimental design considerations

The number of biological replicates is more important than sequencing depth for DE analysis. Biological replicates capture natural variation between samples and increase statistical power to detect genuine expression changes. A minimum of three replicates per condition is recommended; more replicates improve the ability to detect subtle changes.

:::warning
Biological replicates (RNA from distinct samples) are not the same as technical replicates (the same RNA sequenced multiple times). DE analysis requires biological replicates.
:::

Paired-end sequencing improves mapping accuracy and is preferable for DE analysis, but single-end data also works well for gene-level quantification.

## Prepare your samplesheet

Name your samples clearly to reflect the experimental conditions. The `sample` column values are used to derive condition and replicate information for the DESeq2 QC step:

```csv title="samplesheet.csv"
sample,fastq_1,fastq_2,strandedness
CONTROL_REP1,control_rep1_R1.fastq.gz,control_rep1_R2.fastq.gz,auto
CONTROL_REP2,control_rep2_R1.fastq.gz,control_rep2_R2.fastq.gz,auto
CONTROL_REP3,control_rep3_R1.fastq.gz,control_rep3_R2.fastq.gz,auto
TREATMENT_REP1,treatment_rep1_R1.fastq.gz,treatment_rep1_R2.fastq.gz,auto
TREATMENT_REP2,treatment_rep2_R1.fastq.gz,treatment_rep2_R2.fastq.gz,auto
TREATMENT_REP3,treatment_rep3_R1.fastq.gz,treatment_rep3_R2.fastq.gz,auto
```

See the [samplesheet documentation](../samplesheet.md) for the full format specification.

## Run the pipeline

Use `--fasta` and `--gtf` to provide your reference genome and annotation directly. Do not use iGenomes, as those annotations are out of date (see [reference genomes](../reference-genomes.md)):

```bash
nextflow run nf-core/rnaseq \
    --input samplesheet.csv \
    --outdir results \
    --fasta /path/to/genome.fa \
    --gtf /path/to/annotation.gtf \
    -profile docker
```

The default aligner is STAR with Salmon quantification (`--aligner star_salmon`), which produces the DESeq2 object used in the next step. See [alignment and quantification](../alignment-and-quantification.md) for alternative analysis strategies.

## Check your results

After the pipeline completes, verify the output quality before proceeding to DE analysis:

1. **MultiQC report** — Open `results/multiqc/star_salmon/multiqc_report.html` and check for sample-level QC issues (adapter contamination, low mapping rates, high duplication)
2. **PCA plot** — Check `results/star_salmon/deseq2_qc/` for the PCA plot generated by the pipeline. Samples within the same condition group should cluster together.
3. **Sample clustering** — Verify that the hierarchical clustering heatmap shows the expected grouping by condition.

If the QC looks satisfactory, proceed to [DE analysis in R](de-analysis-in-r.md).

## Pipeline output for DE analysis

The key output file for DE analysis is the pre-computed DESeq2 object:

```
results/star_salmon/deseq2_qc/deseq2.dds.RData
```

This file contains the raw count matrix, sample metadata derived from the samplesheet, and a basic design formula. You will refine the metadata and design in the next step of this tutorial.
